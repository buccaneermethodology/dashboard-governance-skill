# Dashboard Methodology

The Dashboard is a cognitive tracking system for an evolving project.

It preserves:

- visibility into explored work
- traceability across implementation and thinking
- optionality for next steps

It is not:

- a pure task list
- a canonical design document
- a final report

## Big Idea

A Big Idea is a long-lived semantic track or implementation direction. It can span many sessions and should not close automatically when one session finishes.

## Session

A Session is a bounded execution or exploration unit. It should have Topic, Scope, Purpose, priority, status, exit criteria, and durable notes.

## Decision

A Decision records a choice that could block or distort multiple sessions if left implicit.
